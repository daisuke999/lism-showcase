# テンプレ配布用：ページを1つの Astro ファイルにマージする手順

このドキュメントは、**開発用の分割構成**（`lp00x.astro` + `src/components/lp00x/**`）を維持したまま、**配布用に1ファイルへまとめた Astro** を別パスに生成するときの指示です。  
Cursor 等のエージェントにマージ作業を依頼する際は、**このファイルを読むよう明示**したうえで作業してください。

※ **命名**: エントリページもコンポーネントディレクトリもマージ成果物ディレクトリも、同一の **`lp00x`**（例：`lp001`）で揃えます。マージ後は `src/pages/merged/lp00x/` に **`index.astro`** と **`_style.css`** を置きます。

---

## 1. 目的

- 完成した **`lp001`** などを、**多くのユーザーがコピーして使えるテンプレート**として配布する。
- 配布形態として **「1つの `.astro` ファイルで完結」** することが必要である。
- **マージ元のページとコンポーネントは開発資産として残す**（上書き・削除しない）。

---

## 2. 用語

| 用語 | 意味 |
|------|------|
| **マージ元** | `src/pages/lp00x.astro` および、そこから import している `src/components/lp00x/**/*.astro`（必要なら同階層の CSS 等） |
| **マージ成果物** | マージ元の内容を1ファイルに集約した `.astro`。**本リポジトリでは `src/pages/merged/lp00x/index.astro`** として置く（マージ元と同じ **`lp00x`** 名のフォルダに **`index.astro`** と空の **`_style.css`**。詳細は **セクション4**） |

---

## 3. 不変条件（必ず守る）

1. **マージ元のファイルを編集しない**  
   - `src/pages/lp00x.astro`  
   - `src/components/lp00x/**`  
   - マージのための変更（import 削除、中身の移動など）は **マージ成果物側のみ** で行う。

2. **マージ成果物は別パスに新規作成（または該当ファイルのみ差し替え）**  
   - マージ元と同じパスに「置き換え保存」しない。

3. **成果物の責務**  
   - ページとして動作する1ファイルに、元ページ相当のマークアップ・スクリプト・スタイルが含まれること。  
   - 外部パッケージ（`lism-css` / `@lism-css/ui` / レイアウト等）の import は、**不要にならない限り維持**する。

---

## 4. 出力先の規約（本プロジェクト）

マージ元が **`lp00x.astro`** のとき、成果物は **`merged` 以下に同名の `lp00x` ディレクトリ** を1つ置く（開発時のエントリと **`merged` 直下をフラットな単一 `.astro` で並べる**構成にはしない）。

| 項目 | 値 |
|------|-----|
| ベースディレクトリ | `src/pages/merged/` |
| **成果ディレクトリ名** | **`lp00x`** … マージ元の **`lp001.astro`** に対応して **`merged/lp001/`** のように **同名** で置く（番号はファイル名からそのまま）。 |
| **ページ本体** | **`src/pages/merged/lp00x/index.astro`** … マージ後の Astro の内容はすべてここへ書く（このファイルが `/merged/lp00x/` のページになる）。 |
| **スタイル用ファイル** | **`src/pages/merged/lp00x/_style.css`** … **必ず同じディレクトリに作成する**。中身は空でよい（プロジェクトや配布先でカスタム CSS を足すためのプレースホルダ）。 |

例：**`lp001`** をマージするときは次の2ファイルを出力する。

```text
src/pages/merged/lp001/index.astro    ← マージ済みの単一ページ
src/pages/merged/lp001/_style.css    ← 空でよい
```

- 初回は `merged` および `merged/lp00x` が無ければ作成してよい。
- **`lp004`** 以降も同じ規約で **`merged/lp004/index.astro`** と **`merged/lp004/_style.css`** とする。

### 本リポジトリ: lp001 の再生成

- マージ元（`src/pages/lp001.astro` と `src/components/lp001/**`）を更新したあと、成果物を作り直す場合は、**再生成は `scripts/emit-merged-page001.mjs`**（または同等の **`scripts/emit-merged-page.mjs`**）を使う。  
  - プロジェクトルートで **`node scripts/emit-merged-page001.mjs`** を実行すると、`src/pages/merged/lp001/index.astro`（および **`_style.css`**）が期待どおり更新される想定とする（スクリプトはマージ元を読み取るのみで、**マージ元ファイルは編集しない**）。

---

## 5. マージ作業で行うこと（エージェント向け）

### マージ成果物の生成方法

マージは次のいずれかで行うこと。

- **手書き** … マークアップ・frontmatter・スタイルをエディタ上で1ファイルに組み立てる。
- **スクリプト** … 複数ファイルを連結する際は、**複数行にまたがる `import` を正しく結合できる**実装にすること（行単位の机上連結は、`import {` の途中で途切れるなど**構文エラー**になりやすい）。

いずれにせよ、成果物の frontmatter は **有効な Astro / TypeScript として解釈できる**状態にする。

1. **マージ元の特定**  
   - 依頼で指定された **`lp00x`** について、`src/pages/lp00x.astro` の frontmatter の import 一覧を把握する。  
   - `@/components/lp00x/` 以下の各 `.astro` を、**依存順を壊さないよう**読み込む（子コンポーネントが更に別ファイルを import している場合は再帰的に辿る）。

2. **単一ファイルへの集約**  
   - 各コンポーネントの **テンプレート部分** を、親子の構造が崩れないように **1つのファイル内**へ移す。  
   - コンポーネント専用だった `---` 内の import / 型 / 定数 / `define:vars` 等は、**重複・衝突を解消**しつち上段の frontmatter に整理する。  
   - コンポーネント名のカスタム要素（例：`<Header />`）は、**インライン化後は不要**ならそのままのタグ構造に展開してよい（可読性と重複のバランスは依頼者の好みに合わせる。原則「1ファイルで完結」が最優先）。

3. **import の整理**  
   - 同一モジュールからの import は **まとめてよい**（例：`lism-css/astro` の複数コンポーネント）。複数行の `import { ... } from '…'` は **結合後も1つの import 文として閉じる**こと（上記「生成方法」を参照）。  
   - マージにより不要になった `@/components/lp00x/...` の import は **成果物から削除**する。  
   - **`DemoLayout` / `lism-css/astro` / `@lism-css/ui/astro`** は **削除せず残す**（詳細は **セクション6**）。  
   - `@/` エイリアスや `lism-css` のパスは、プロジェクトの既存の書き方に合わせる。

4. **`<style>`**  
   - 元ページ・各コンポーネントに複数の `<style>` がある場合は、**セレクタの衝突に注意**しつつ、**1ファイル内の `<style>` に統合**してよい。  
   - **`@layer lism-custom` に置かれている CSS は、成果物では1つの `@layer lism-custom { ... }` にまとめる**（マージ元が複数コンポーネント・複数 `<style>` に分かれていても、同一レイヤーへ統合する）。  
   - その他の `@layer` やスコープの意図があれば可能な限り維持する。

5. **ルーティング**  
   - `src/pages/merged/lp001/index.astro` は **別 URL** になる（例：**`/merged/lp001/`** …末尾スラッシュの有無は Astro の設定に従う）。  
   - マージ作業では **開発用エントリのルートを変えない**（マージ元の `src/pages/lp001.astro` は触らないため）。開発時の URL は **`/lp001`** のような **`lp00x` に対応するパス** を参照する。

---

## 6. import の扱い（固定方針）

マージ成果物では、次の **外部 import はそのまま残す**（マージ元と同様の import 文でよい）。配布先でも、プロジェクトがこれらの依存を持てば **そのまま動作する**前提とする。

| 残す import の例 | 備考 |
|------------------|------|
| `DemoLayout`（`@/layouts/DemoLayout.astro` 等） | レイアウトはファイル分割のまま |
| `lism-css/astro` | Lism のプリミティブ・`Lism` 等 |
| `@lism-css/ui/astro` | UI コンポーネント（Button 等） |

**インライン化する対象**は、主に **`@/components/lp00x/**` から import しているページ専用コンポーネント**である。これらを1ファイル内に展開し、当該 `@/components/lp00x/...` の import は成果物から削除する。

上記以外の import（プロジェクト独自の `layouts` / `utils` 等）がある場合は、マージ元に合わせて残すか依頼者に確認する。

## 7. 完了前チェックリスト

- [ ] マージ元の `src/pages/lp00x.astro` と `src/components/lp00x/**` に **差分が無い**（編集していない）。  
- [ ] **`src/pages/merged/lp00x/index.astro`** が存在し、ビルド可能な構文になっている。  
- [ ] 同じ **`lp00x`** ディレクトリに **`_style.css`** がある（空でよい）。  
- [ ] 未使用の import・未使用の変数が可能な範囲で整理されている。  
- [ ] **セクション6**どおり、`DemoLayout` / `lism-css/astro` / `@lism-css/ui/astro` は import のまま残し、ページ専用コンポーネントはインライン化している。  
- [ ] **`@layer lism-custom` のスタイルが1つの `@layer lism-custom` にまとまっている**（重複レイヤーブロックになっていない）。  
- [ ] frontmatter の **複数行 `import` が壊れていない**（必要なら手書き or 正しい連結スクリプトで生成したことを確認済み）。

推奨：作業後に **`npm run build`** を実行し、エラーがないことを確認する（実行可能なら）。

## 8. 依頼時のチャット例

### 単一ページ（例：lp001）

```text
docs/merge-single-file-template-pages.md の内容に従い、lp001 を1つの Astro ファイルにまとめた成果物を
src/pages/merged/lp001/index.astro として作成し、同じディレクトリに空の _style.css を置いてください。
マージ元のファイルは編集しないでください。
```

`lp004` などは `lp001` を読み替える。

### 複数ページを同時にマージする（例：lp001・lp002・lp003）

```text
docs/merge-single-file-template-pages.md の内容に従い、lp001・lp002・lp003 をそれぞれ1つの Astro ファイルにまとめた成果物を作成してください。
各ページごとに次を出力してください（マージ元のファイルは編集しないこと）。
・lp001 → src/pages/merged/lp001/index.astro と src/pages/merged/lp001/_style.css（空）
・lp002 → src/pages/merged/lp002/index.astro と src/pages/merged/lp002/_style.css（空）
・lp003 → src/pages/merged/lp003/index.astro と src/pages/merged/lp003/_style.css（空）
```

本リポジトリでは **`node scripts/emit-merged-page.mjs 001`** のようにページ ID を変えて **001・002・003 を順に実行**してもよい（各実行が対応する **`lp00x`** を生成する）。

## 9. メンテナンス

- プロジェクトのパス規約や import 方針が変わったら、**このファイルの「4. 出力先」「5. マージ作業」**を更新する。  
- 配布パッケージの置き場所（npm / 別リポジトリ等）が決まったら、「成果物をどこへコピーするか」を追記するとよい。
